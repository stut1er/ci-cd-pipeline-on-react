## О проекте

Полноценный CI/CD пайплайн для React + Vite приложения с автоматическим тестированием и деплоем.

**Технологии:** React, Vite, GitHub Actions, Netlify.

## Что происходит при push в master

GitHub Actions автоматически запускает 5 этапов проверки: один установочный + четыре проверочных(параллельно)

| № | Этап | Инструмент | Что проверяет |
|---|------|------------|----------------|
| 1 | **Type Check** | TypeScript (`tsc --noEmit`) | Корректность типов данных |
| 2 | **Lint** | ESLint | Стиль кода, потенциальные ошибки |
| 3 | **Unit Tests** | Vitest | Логику компонентов в изоляции |
| 4 | **E2E Tests** | Playwright | Полный пользовательский сценарий в браузере |
| 5 | **Storybook Tests** | Storybook Test Runner | Рендер компонентов без ошибок |

 **Все этапы должны пройти успешно** → автоматический деплой на Netlify

### Unit тесты (Vitest) – 17 тестов

**Counter (7 тестов)**
- Рендер с разными начальными значениями
- Клики на кнопках (+1, -1, Сброс)
- Корректность изменения состояния
- Отображение чётности числа
- Соблюдение минимального и максимального значений

**TodoList (4 теста)**
- Рендер заголовка и пустого списка
- Добавление новой задачи
- Переключение статуса задачи
- Удаление задачи

**UserCard (5 тестов)**
- Рендер имени и email
- Отображение роли (по умолчанию и переданной)
- Рендер аватара при наличии

**App (1 тест)**
- Проверка рендера главного компонента

### E2E тесты (Playwright) – 3 сценария

1. **Счётчик** – полный цикл работы (+1, -1, Сброс, проверка чётности)
2. **TodoList** – добавление, выполнение и удаление задач
3. **Интеграционный** – взаимодействие нескольких компонентов на странице

### Storybook тесты – 6 историй

**Counter**
- Стандартный счётчик
- Счётчик с начальным значением 10
- Счётчик с ограничениями (min: 0, max: 10)

**TodoList**
- Пустой список задач
- Список с задачами 

**UserCard**
- Карточка с аватаркой
- Карточка без аватарки
